# OO - Orientação Objeto

A **Programação Orientada a Objetos (POO)** é um paradigma de programação que organiza o código em torno de **objetos**, que são instâncias de **classes**. Ela facilita o desenvolvimento de software complexo, promovendo reutilização, modularidade e manutenção simplificada. Os principais conceitos são:

## 1. Classe e Objeto

- **Classe**: Modelo ou estrutura que define atributos (dados) e métodos (funções).  
- **Objeto**: Instância de uma classe; contém valores concretos e comportamentos.

## 2. Encapsulamento

Protege os dados internos do objeto, permitindo o acesso controlado por meio de métodos (como *getters* e *setters*). Isso evita modificações indevidas e promove a integridade dos dados.

## 3. Herança

Permite que uma classe (subclasse) herde atributos e métodos de outra (superclasse). Promove reutilização de código e facilita a extensão de funcionalidades.

## 4. Polimorfismo

Permite que objetos de diferentes classes sejam tratados de forma uniforme, principalmente ao usar métodos com o mesmo nome, mas comportamentos distintos. Pode ser de dois tipos:
- **Polimorfismo de Sobrecarga**: Mesmos métodos com diferentes parâmetros na mesma classe.
- **Polimorfismo de Sobrescrita**: Uma subclasse redefine um método da superclasse.

## 5. Interface

Define um conjunto de métodos que uma classe deve implementar, sem especificar sua implementação. Facilita a criação de contratos e garante que diferentes classes sigam um padrão comum.

Esses conceitos trabalham juntos para criar software modular, reutilizável e de fácil manutenção.
